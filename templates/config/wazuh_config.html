แก้ไขการตั้งค่า
1 เปิดใช้ Vulnerability Detector จำเป็น

ต้องเปิดบน Wazuh Manager เพื่อให้ระบบ scan CVE และส่งข้อมูลไปยัง Indexer

แก้ไขไฟล์: /var/ossec/etc/ossec.conf
<ossec_config> <vulnerability-detector> <enabled>yes</enabled> <interval>5m</interval> <min_full_scan_interval>6h</min_full_scan_interval> <run_on_start>yes</run_on_start> <!-- NVD feed --> <provider name="nvd"> <enabled>yes</enabled> <update_interval>1h</update_interval> </provider> <!-- Red Hat / Ubuntu / Debian / Amazon Linux --> <provider name="redhat"><enabled>yes</enabled></provider> <provider name="debian"><enabled>yes</enabled></provider> <provider name="ubuntu"><enabled>yes</enabled></provider> <provider name="alas"><enabled>yes</enabled></provider> </vulnerability-detector> </ossec_config>
# restart หลังแก้ config sudo systemctl restart wazuh-manager # ตรวจสอบ sudo /var/ossec/bin/wazuh-control status | grep vuln
2 เปิดใช้ Syscollector บน Agent แนะนำ

Syscollector เก็บข้อมูล port, process, package บน agent — จำเป็นสำหรับ vulnerability scan

แก้ไขบน Agent: /var/ossec/etc/ossec.conf
<wodle name="syscollector"> <disabled>no</disabled> <interval>1h</interval> <scan_on_start>yes</scan_on_start> <hardware>yes</hardware> <os>yes</os> <network>yes</network> <packages>yes</packages> <ports all="no">yes</ports> <processes>yes</processes> </wodle>
3 ติดตั้ง Custom Integration (Real-time webhook) optional

ส่ง alert มายัง Dashboard แบบ real-time เมื่อ Wazuh ตรวจพบ — ไม่ต้องรอ polling

ขั้นตอนที่ 1 — copy script
# บน Wazuh Manager sudo cp /path/to/soc-dashboard/wazuh-integration/custom-soc-dashboard \ /var/ossec/integrations/ sudo chown root:wazuh /var/ossec/integrations/custom-soc-dashboard sudo chmod 750 /var/ossec/integrations/custom-soc-dashboard
ขั้นตอนที่ 2 — แก้ URL ใน script
DASHBOARD_URL = "{{ dashboard_url }}" API_TOKEN = "<token จาก /admin/ → Auth Token>"
ขั้นตอนที่ 3 — เพิ่มใน ossec.conf (Manager)
<integration> <name>custom-soc-dashboard</name> <level>3</level> <alert_format>json</alert_format> </integration>
sudo systemctl restart wazuh-manager
4 Active Response (สำหรับ Auto Block IP) อนาคต

ถ้าต้องการให้ Dashboard สั่ง block IP บน agent อัตโนมัติ ต้องเปิด Active Response

ossec.conf บน Manager
<active-response> <disabled>no</disabled> <command>firewall-drop</command> <location>local</location> <rules_id>100001</rules_id> <timeout>600</timeout> </active-response> <command> <name>firewall-drop</name> <executable>firewall-drop</executable> <timeout_allowed>yes</timeout_allowed> </command>
5 รับ Syslog จาก Switch / Firewall อนาคต

ให้ Wazuh Manager หรือ Agent รับ syslog UDP 514 จาก network device

ossec.conf บน Manager / logrelay agent
<!-- รับ syslog UDP 514 --> <remote> <connection>syslog</connection> <port>514</port> <protocol>udp</protocol> <allowed-ips>192.168.0.0/16</allowed-ips> </remote> <!-- ตั้งค่า switch/firewall ให้ส่ง syslog มาที่ IP ของเครื่องนี้ port 514 -->
